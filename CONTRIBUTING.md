# Contributing

Thanks for your interest in contributing to this theme!

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Make your changes
5. Test with a local Vertex CMS (`@koehler8/cms`) project to verify your changes render correctly

## Submitting Changes

1. Create a feature branch from `main`
2. Commit your changes with clear, descriptive messages
3. Push to your fork and open a pull request
4. Describe what you changed and why

## Guidelines

- Keep token names consistent with the [theme schema](https://github.com/koehler8/cms)
- Test color contrast for accessibility (WCAG AA minimum)
- Scope CSS selectors with `[data-site-theme]` to avoid cross-theme bleed
- Do not introduce JavaScript — themes are CSS/config only

## Reporting Issues

Use [GitHub Issues](https://github.com/koehler8/cms-theme-swamp/issues) to report bugs or request features.
